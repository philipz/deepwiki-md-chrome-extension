Purpose and Scope

This document details the main components that make up the QCustomPlot library's architecture. These components form the foundation of the library's plotting and visualization capabilities. For information about how data flows through these components, see Data Flow.

Component Overview

QCustomPlot is structured around several key components that work together to create a flexible plotting system. Each component handles specific aspects of plot creation, data representation, and visualization.

Sources: README.md

QCustomPlot Widget

The QCustomPlot class is the main widget that serves as the entry point for all plotting functionality. It inherits from Qt's QWidget class, making it easily integrable into any Qt application.

Key responsibilities:

  • Management of plot elements (graphs, axes, legends)
  • Event handling (mouse/keyboard interactions)
  • Coordinating drawing operations
  • Managing data structures
  • Providing the interface for plot customization
  • Handling export functionality

Sources: README.md

Layer System

The Layer System provides depth control for visual elements in the plot. It ensures elements are drawn in the correct order (background first, foreground elements last).

Key features:

  • Hierarchical organization of plot elements
  • Control over drawing order
  • Selective visibility control
  • Performance optimization through selective redrawing

Sources: README.md

Axes System

The Axes System provides the framework for mapping data values to screen coordinates and displaying reference information.

Axis Rectangle (QCPAxisRect)

Defines a rectangular area in which plotting occurs, containing multiple axes (typically at least one x-axis and one y-axis).

Axis Ticker (QCPAxisTicker)

Controls how tick marks and labels are generated and placed along an axis.

Axis Types

QCustomPlot supports various axis configurations:

  • Linear axes (default)
  • Logarithmic axes
  • Date/time axes
  • Custom-scaled axes

Sources: README.md

Plottable Objects

Plottables are objects that represent data visually on the plot. They derive from the abstract base class QCPAbstractPlottable.

QCPGraph

Used for creating line and scatter plots. Connects data points with lines and/or represents them as symbols.

QCPCurve

Similar to graphs but specialized for parametric curves where points are not necessarily equidistant on the x-axis.

QCPBars

Used for creating bar charts to visually compare categorical data.

QCPStatisticalBox

Implements box plots to display statistical distributions with quartiles, medians, and outliers.

QCPColorMap

Displays 2D data as a color-coded map where color represents a data value.

Sources: README.md

Item Objects

Items are visual elements that can be added to plots but are not directly tied to data series. They're used for annotations, highlighting, and additional information.

QCPItemText

For adding text labels and annotations.

QCPItemLine

For adding custom lines to plots.

QCPItemArrow

For adding directional indicators.

QCPItemBracket

For grouping or highlighting regions of a plot.

Sources: README.md

Layout System

The Layout System manages the arrangement of elements within the plot. It provides a flexible way to organize axes, plots, and other elements.

Layout Elements

The base components that can be arranged within layouts:

  • QCPLayoutElement: Abstract base class
  • QCPAxisRect: Contains axes and the plotting area
  • QCPLegend: Displays legend items
  • QCPTextElement: Displays text (titles, labels)

Layout Types

  • QCPLayoutGrid: Arranges elements in a grid
  • QCPLayoutInset: Places elements at specific positions relative to a parent element

Sources: README.md

Export System

QCustomPlot includes a comprehensive export system for generating high-quality output in various formats.

Key features:

  • Vector export (PDF)
  • Raster export (PNG, JPG, BMP)
  • Customizable resolution and scaling
  • Publication-quality output

Sources: README.md

Data Handling

QCustomPlot uses specialized containers and classes to efficiently manage and process data.

Sources: README.md

Interaction Model

QCustomPlot implements a comprehensive interaction system allowing users to interact with plots through mouse and keyboard events.

Sources: README.md

Summary

The core components of QCustomPlot work together to create a flexible and powerful plotting system. The main QCustomPlot widget coordinates all other components, while specialized classes handle specific aspects of plotting and visualization. This modular architecture allows for extensive customization while maintaining ease of use for common plotting tasks.
